**The Scenario:**
You have a user who keeps trying to run `sudo` commands they aren't allowed to run. You want to see these "Unauthorized" attempts.

**The Question:** 
By default, where does `sudo` log these "Lecture" messages and unauthorized attempts? (Give the full path for a Debian/Ubuntu system). Also, is there a sudoers setting to make `sudo` send an **email** to root every time someone fails a sudo attempt?

**The answer:**
Sudo has built-in mail functions. To send an email to root on every failure, you add these to your `sudoers` file:

Code snippet

```bash
Defaults mail_always 
Defaults mailto="root@localhost"
```

When someone fails a sudo attempt, the system uses the local mail agent (like Postfix or Exim) to alert the admin immediately.
