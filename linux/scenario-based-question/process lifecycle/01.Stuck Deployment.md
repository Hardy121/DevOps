# The Stuck Deployment
You are deploying a new version of a Python-based background worker on a Linux staging server. You’ve updated the code and triggered a restart script. However, you notice that the application isn't picking up the new configuration changes.

Upon investigation, you realize that the old process didn't actually stop; it’s still running in the background, holding onto the network port, and preventing the new version from starting correctly.

# Your Task
**Identify*:
Which command would you use to find the PID (Process ID) of a process named worker_app, and how would you verify its current state (e.g., Running, Sleeping, or Defunct)?

**Act*:
You try to stop it gracefully using kill <PID>, but the process ignores you and stays alive. What is the specific command (including the signal) you would use to force the kernel to kill this process immediately?

**Cleanup*:
After killing it, you see the process is still listed in top but marked as <defunct>. What is this state commonly called in DevOps, and what is the standard way to clear it from the process table if it doesn't disappear on its own?
