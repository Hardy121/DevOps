# The Scenario
In DevOps, you often find yourself in a situation where you start a script (like a long rsync data migration or a terraform apply) and realize, "Wait, this is taking way longer than I thought, and I need my terminal back to check something else!"

You don't want to kill it and start over, but you can't just close the terminal.

# Your Task
Suspend: You have a script running in the foreground right now. Which keyboard shortcut do you press to "pause" it and send it to the background in a "Stopped" state?

Resume: Now that it's stopped, you see it listed when you type jobs. Which command do you use to make it start running again in the background (so you can still type other commands)?

Bring Back: After a few minutes, you want to see the progress again. Which command brings that specific background job back to the foreground?

# The Answer
1. `CTRL + z`
2. `bg`
3. `fg`
