# The Scenario
You are on a Jenkins Master node. The root partition / is at 100%, and the CI/CD pipeline is failing because it can't write temporary logs.
You run lsblk and see:
sda (System Disk) is 50GB.
sda1 is the / partition (EXT4), and it is not using LVM (it's a standard partition).
You just added a new 100GB disk at /dev/sdc.

# The Question
Since the root partition is not LVM, you can't easily "expand" it with the new disk. A common DevOps fix is to "Mount a new disk to a specific heavy directory."

How would you:
1. Format the new disk /dev/sdc with an ext4 filesystem?
2. Mount it temporarily to a folder called /mnt/new_logs?
3. Bonus: What file do you need to edit to make sure this disk stays mounted after a reboot?

# The Answer
1. `sudo mkfs.ext4 /dev/sdc`
2. `sudo mount /dev/sdc /mnt.new_logs`
3. `/etc/fstab`
