# The Scenario 

You are on a brand new Ubuntu server. You need to switch to the root user account permanently to do some heavy maintenance. You try `su -`, but it asks for a "Password." You try your own user password, and it fails. You try to leave it blank, and it fails. You realize you don't know the root password because you were never given one.

# The Question:
In modern distributions like Ubuntu, the root account is "locked" (it has no password). What is the **internal sudo command** that allows you to open a root login shell using *your own* password instead of root's?

**The Answer:** `sudo -i` 

Why `sudo -i`  and not `sudo su -` 

`sudo su -` starts two separate processes (sudo and then su). `sudo -i` tells the `sudo` binary itself to look at the target user's (root) login shell and start it directly. It’s cleaner and logs the action more accurately in the system audit logs.
