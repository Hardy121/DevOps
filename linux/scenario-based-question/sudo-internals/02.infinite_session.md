# Scenario:
A junior admin is worried about security.
They noticed that after they type their password for `sudo` once,
they can run `sudo` commands for the next several minutes without being asked for a password again.
They want to know where this "proof of identity" is stored on the disk so they can delete it manually if they leave their desk.

# Question:
What is the full path to the directory where `sudo` stores these session "tickets" (timestamps), and what command allows a user to "kill" their own ticket immediately without deleting files manually?

# Answer:
The full path to the directory where sudo stores session ticket is : `/run/sudo/ts/`
the command that allows a user to kill their own ticket immediately without deleting files is:  `sudo -k`
