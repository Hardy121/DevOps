# Scenario:
You just installed a custom tool in `/opt/special/bin/mytool`. 
You added `/opt/special/bin` to your `$PATH` in your `.bashrc`, and it works perfectly when you run it as a normal user.
However, when you try to run it with `sudo mytool`, the system screams:
`sudo: mytool: command not found`

Even if you check `echo $PATH` right before running sudo, the directory is there.

# Question:
Why can't `sudo` find the binary, and what is the specific security "internal" in the `sudoers` file that causes this behavior?

# Answer: 
sudo can't find binary because sudo doesnot trust blindly on whatever you add in $PATH it can be tricky to run.
so, sudo reset the Environment variable through the "env_reset" in sudoers file.
