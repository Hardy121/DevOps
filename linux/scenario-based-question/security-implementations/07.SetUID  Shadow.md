# The Scenario:

In the **User Space** layer, there are special permissions called **SUID (Set User ID)**. When a file has this bit set, it runs with the permissions of the **owner** (usually root), no matter who executes it.

As a DevOps engineer, "shadow" SUID binaries are a huge security risk. If a hacker finds a script with SUID root that they can edit, they have just won the game.

# The Question:

You are auditing a new server image. You need to find **all files** in the `/usr/bin` directory that have the **SUID bit** set (so you can make sure no unnecessary "backdoors" exist).

What is the specific `find` command flag used to look for files with the `4000` (SUID) permission bit?**

# The Answer:

`find /usr/bin -type f -perm  -4000`
