# The Scenario  
In your DevOps pipeline, you have a cleanup script at /usr/local/bin/cleanup_logs.sh. Traditionally, you would use a Cron job to run this. However, your team is moving toward a systemd-native infrastructure.

The Benefit: Unlike Cron, systemd Timers allow you to see exactly when the task last ran, view its logs in journalctl, and even set resource limits (like we did with CPU/RAM) so the cleanup script doesn't slow down the main API.

Your Goal:
Create a timer to run your cleanup script every day at 3:00 AM.

# The Question
1. To set up a timer, you actually need two files in /etc/systemd/system/. One is the .timer file. What is the extension of the second file that actually tells systemd what script to run?

2. Inside the [Timer] section, which directive do you use to define the specific time 

3. If you want the timer to run 15 minutes after the server boots up which directive would you use?

# The Answer
1. `service`
2. `OnCalender=*-*-* 03:00:00`
3. `OnBootSec=15M`
