**The Scenario:**
You have a script that needs to run a command as the user `www-data` (the web server user), not as `root`. You are logged in as yourself. You try to run: `sudo -u www-data whoami`
It works! It says `www-data`.

However, you then try to run a script that relies on the web server's environment variables (like `WEB_ROOT`), but the script fails because those variables are missing.

**The Question:** 

By default, `sudo` environment variables are "reset." What **internal sudo flag** do you add to your command to ensure you not only run as `www-data` but also load `www-data`'s specific environment and home directory?

**The Answer:** 
To get **`www-data`'s** environment (like their specific `$HOME` or variables defined in their profiles), you should use **`sudo -u www-data -i`**.
