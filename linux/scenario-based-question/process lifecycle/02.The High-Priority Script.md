# The Scenario
The High-Priority Script
Imagine you are running a heavy database migration script on a server that is already under high load. You don't want the migration to steal all the CPU and make the website slow for users, but you also don't want it to take 10 hours.

# The Question:

1. How would you start your script (let's call it migrate.sh) with a lower priority so the kernel gives preference to other tasks?
2. If the script is already running, which command would you use to change its priority "on the fly" to make it even less of a resource hog?
3. In Linux priority levels (Nice values), does a higher number (e.g., +19) mean high priority or low priority?

# The Answer:
1. ``nice -n 10 ./migrate.sh``
2. ``renice 15 <PID>``
3. Higher value means Lower priority
