# The Scenario:
You have a user named `dev_user` who needs to edit a specific configuration file at `/etc/myapp/config.conf`which is owned by root.
You don't want to give them full root access, so you give them this rule in `/etc/sudoers`:
`dev_user ALL=(ALL) /usr/bin/vim /etc/myapp/config.conf`

Within 5 minutes, `dev_user` has gained a full root shell and changed the system password.


# The Question:
How did using a standard text editor like `vim` via `sudo` allow the user to break out into a root shell (the "Escape" technique), and what internal `sudo` command/tool should you have used instead to let them edit the file safely?

# The Answer:

The Escape technique to root from vim is. press Esc, then type :!bash. and the user convert to the root user.

To prevent that, do not give vim access, give only                                       

`sudoedit /etc/myapp/config.conf`
