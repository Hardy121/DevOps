**The Scenario:** 

You want to allow a user to run a custom script located at `/home/user/scripts/deploy.sh`. You add this to sudoers:
`user ALL=(ALL) /home/user/scripts/deploy.sh`
The user is clever. They realize they own that directory. They delete the `deploy.sh` file and create a *new* one with the same name that contains `rm -rf /`.

**The Question:**

What is the **Internal Security Rule** regarding directory permissions that you should follow to prevent a user from swapping out a sudo-authorized script for a malicious one?

**The Answer:**

Any script allowed in `sudoers` **MUST** be owned by `root` and located in a directory that the user **cannot write to** (like `/usr/local/bin/`). If the user owns the folder, they can delete and replace the script, bypassing sudo security entirely.
