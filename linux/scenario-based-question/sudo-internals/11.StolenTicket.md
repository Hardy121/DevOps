**The Scenario:**

You are working in a coffee shop. You run `sudo apt update`, type your password, and finish your work. You walk away for 2 minutes to get a refill. A hacker sits at your laptop. They don't know your password, but they type `sudo useradd hacker_account` and it **works immediately** without asking for a password.

**The Question:**

Internally, `sudo` uses a "ticket" system.
1. How long does this ticket stay valid by default (on most systems)?
2. What is the name of the **Parameter** in the `/etc/sudoers` file you would change to set this time to `0` (forcing a password every single time)?

**The Answer:**

The default ticker is 5 minutes or 15 minutes in the system 
To make it 0 (forcing password every time)
add this in sudoers file
`Defaults timestamp_timeout=0`
