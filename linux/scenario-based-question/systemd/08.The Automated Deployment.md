# The Scenario
You are building a CI/CD pipeline. Your goal is to deploy a new Node.js microservice. To make this production-ready, you need to combine everything we’ve practiced.

The Task:
You have a script that creates a service file at /etc/systemd/system/node-app.service. You need to ensure:

The app runs as the node-user.

It restarts automatically if it crashes.

It cannot use more than 1GB of RAM.

It logs everything so you can debug later.

# The Questions
1. The Mystery Error: You run systemctl start node-app, but it fails. You run journalctl -u node-app and see: Executable path is not absolute. What did you likely forget to do in the ExecStart= directive of your service file?

2. The Automation Check: In a script, if you want to enable the service and start it in one single command (instead of two), what flag do you add to systemctl enable?

3. The Performance Audit: Your boss asks for a graph showing the boot sequence to see if the new node-app is slowing down the server. What systemd-analyze command generates an SVG file of the entire boot process?

# The answers
1. I forgot to add exact path of the script that needes to run when service start on `ExecStart=`
2. ` systemctl enable --now node-app.serivce`
3. `systemd-analyze plot > boot_analysis.svg`
