# The Scenario 
You are logged in as a standard user. You need to add a new DNS server to the system configuration. You run:
`sudo echo "nameserver 8.8.8.8" >> /etc/resolv.conf`
The terminal immediately replies:
`-bash: /etc/resolv.conf: Permission denied`

You are confused because you used `sudo`, and you definitely have permissions to run `echo` as root.

# The Question:

Why did this fail, and what is the **one-line command** using a pipe (`|`) and a specific utility to fix this without opening an editor?

#  The Answer:

In `sudo echo "nameserver 8.8.8.8" >> /etc/resolv.conf`, the `sudo` only applies to the `echo` command. The `>>` (redirection) is handled by **your current shell** (bash/zsh). Your shell tries to open `/etc/resolv.conf` to write to it before `sudo` even starts, and since your shell isn't root, it fails.

IMPORTANT NOTE: sudo works only on the commands that live inside /bin /usr/bin or usr/local/bin. If any command is built into the shell, sudo can not access it.

# The Command:
```bash
echo "nameserver 8.8.8.8" | sudo tee -a /etc/resolv.conf
```

`echo` runs as you. The output is "piped" to `tee`. `tee` is running with `sudo` (root), so it has the power to write to the file. The `a` flag stands for **append**, which is the same as `>>`.
