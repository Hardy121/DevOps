# The Scenario 
A developer asks you to attach a brand new 100GB EBS volume (cloud disk) to a Linux instance for a new Prometheus database. You’ve attached the hardware via the cloud console, but when you run df -h, the new space doesn't show up.

# The Question:
What commands do you use to:
1. Find the name of the new, raw "unformatted" disk?
2. Initialize it as an LVM Physical Volume (PV)?
3. Create a new Volume Group (VG) named vg_prometheus using that disk?

# The Answer
i will found it via 
`lsblk`

then make partitions of it through `fdisk`
and initialize through `sudo pvcreate /dev/sdb1`
and then create group `sudo vgcreate vg_prometheus`
