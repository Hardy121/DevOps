**The Scenario:**

You were manually editing `/etc/sudoers` (without using `visudo`), and you made a typo. You also accidentally changed the permissions of the file. Now, every time you try to run `sudo`, you get this error:
`sudo: /etc/sudoers is world writablesudo: no valid sudoers sources found, quitting`

You are logged in as a standard user. You cannot use `sudo` to fix the file because `sudo` refuses to run as long as the config file is "insecure" (world-writable). You don't know the root password.

**The Question:**

 Since `sudo` is broken and you aren't root, how do you fix the permissions of that file? (Hint: Think about a specific `pk...` command that uses a different privilege framework than sudo).

**Answer:**

# With Reboot

Reboot and enter Recovery Mode, then gain root privilege after then edit sudoers file and make as it was.

# Without Reboot

If the system has **PolicyKit** installed, you can sometimes use:
`pkexec chmod 0440 /etc/sudoerspkexec` is the "Graphical/Modern" cousin of sudo. If you are in the `admin` or `sudo` group, it will prompt you with a GUI-style pop-up. If it works, you fix the file without the downtime of a reboot!
