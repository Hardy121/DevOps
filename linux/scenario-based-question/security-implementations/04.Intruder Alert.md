# The Scenario:

Now that the front door is locked (UFW), we need a "Security Camera" to see who is trying to break in. In DevOps, monitoring logs is how we detect attacks or misconfigurations in our CI/CD pipelines.

On Ubuntu, the **auth.log** tracks every login attempt, sudo command, and SSH connection.

# The Question:

You suspect someone is trying to brute-force your SSH password.

1. **In which specific directory and file name** are the authentication logs stored on Ubuntu?
2. **What command** would you use to watch this file in "real-time" so you can see login attempts as they happen?

# The Answer:
`/var/log/auth.log`
`tail -f /var/log/auth.log`
