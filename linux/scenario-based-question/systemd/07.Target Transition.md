# The Scenario
Your API is running on a fleet of cloud servers. To save money and improve performance, you realize these servers don't need a GUI (Graphical User Interface). They are currently booting into a mode that starts X11 or Wayland (the desktop environment), which wastes about 1GB of RAM.

In DevOps, we want our servers to be "headless"—running only the CLI (Command Line Interface) and network services.

Your Goal:
Change the system's "Runlevel" (now called Targets in systemd) so that it only boots into the multi-user, non-graphical environment.

# The Question
1. What is the name of the Target that represents a standard multi-user CLI environment (no GUI)?
2. What systemctl command do you use to check what the current default boot target is?
3. What systemctl command do you use to permanently change the default boot target so the server never loads the GUI again, even after a restart?

# The Answer
1. `multi-user.target`
2. `systemctl get-default`
3. `systemctl set-defualt multi-user.target`
