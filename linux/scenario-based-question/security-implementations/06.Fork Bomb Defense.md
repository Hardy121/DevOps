# The Scenario:

You are managing a shared build server where multiple developers run their CI/CD scripts. One developer's script has a bug (a "fork bomb") that starts creating thousands of processes, eating up all the CPU and RAM, and locking you out of the server.

In DevOps, we use **User Limits** to ensure no single user or process can hog all the system resources.

# The Question:

1. Which specific configuration file in `/etc/security/` do you edit to set permanent limits (like maximum number of processes or file size) for a specific user?2. What is the command you can run in your current shell to see your *current* limits (like "open files" or "max user processes")?**

# The Answer: 

Make changes to limit.conf
`username soft nproc 20`
`username hard nproc 30`
and after that view curreny limits by
`ulimit -a`
