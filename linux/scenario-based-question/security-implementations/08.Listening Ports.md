# Scenario:

We are finishing the 10 scenarios at the **Network Layer**. Before you even set up a firewall (UFW), you need to know exactly what is "listening" on your server.

In a DevOps pipeline, you might have a "Health Check" script that verifies if your application started correctly and is bound to the right port.

# The Question:

You just deployed a containerized app that is supposed to be listening on port **3000**. You want to check the server's network statistics to confirm:

1. Which process is using the port.
2. The port number (not the service name like 'http').
3. Only listening sockets.

What is the `ss` (Socket Statistics) command and its flags (the common "alphabet soup" of letters) to show this?**

# The Answer:

`ss -tulnp`

**t**: TCP sockets.
**u**: UDP sockets.
**l**: **Listening** sockets (ignores active connections, focusing on the "server" ports).
**n**: **Numeric** (shows "80" instead of "http"—much faster and more accurate for debugging).
**p**: **Process** (shows which PID/Program is actually holding the port).
