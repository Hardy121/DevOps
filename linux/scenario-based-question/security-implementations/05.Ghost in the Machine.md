# Scenario
Sometimes, knowing *who* logged in isn't enough. You need to know **exactly what they did**. Imagine a sensitive configuration file, like `/etc/nginx/nginx.conf`, was changed, and now the production site is down. Nobody is admitting to it.

In a high-security DevOps environment, we use a tool called **`auditd`** (the Linux Auditing System). It doesn't just log logins; it logs **system calls**.

# The Question:

You want to set up a "watch" on the `/etc/passwd` file so that any time a user tries to write to or modify it, a record is created in the audit logs.

1. **What is the name of the command** used to add a temporary audit rule?
2. **What flag** do you use to specify the **path** of the file you want to watch?

*(Hint: The command name is very similar to the service name `auditd`, and the path flag is just the first letter of "path".)*

# The Answer

Think of `auditd` as a **GoPro camera** strapped to a specific file. Even if a user has `sudo` and deletes a file, `auditd` will record exactly *who* did it and *when*.

### **The Answer: `auditctl`**

1. **The Command:** `auditctl` (The control tool for the audit daemon).
2. **The "Watch" Command:** To watch `/etc/passwd` for any **w**rites or **a**ttributes changes, you would run:
`sudo auditctl -w /etc/passwd -p wa -k passwd_changes`
- **`w`**: Stands for **Watch** (the path to the file).
- **`p wa`**: Stands for **Permissions** (watch for **w**rites and **a**ttribute changes like `chmod`).
- **`k`**: Stands for **Key** (a custom label like "passwd_changes" so you can search for it easily later).
