# The Scenario
You are managing a set of Docker containers running a custom Node.js application. One of the containers starts acting up. You log into the host machine and see a process that has a Parent Process ID (PPID) of 1.

Earlier, we talked about Zombies (dead processes waiting for a parent). Now, we are looking at Orphans.

# Your Task 
Definition: If a parent process (like a web server) crashes or is killed, but its child process (like a database cleanup task) keeps running, what is that child process called?

The "Adoptive Parent": In Linux, when a process loses its parent, it is immediately "adopted" by a system process so it doesn't just wander aimlessly. What is the name (or PID) of this "ancestor" process that adopts all orphaned children?

DevOps Context: In Docker, if your main application (PID 1) isn't designed to "reap" (clean up) processes, your container can fill up with zombies. There is a specific flag you can add to docker run to add a tiny "init" process to handle this. Do you know what that flag is?

# The Answer 
1. Orphan
2. systemd or PID 1
3. `docker run --init my-app-image`
