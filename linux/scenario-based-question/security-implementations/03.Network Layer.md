# Scenario
You have just finished setting up a **Jenkins** automation server on an Ubuntu instance. Jenkins runs on port `8080`. By default, Ubuntu has all ports open or the firewall is off. You want to lock down the server so it only allows:

1. **SSH** (Port 22) so you can manage it.
2. **Jenkins** (Port 8080) so the team can use it.
3. **Deny everything else.**

# The Question:

- *What are the three `ufw` commands you would run to:
1. Enable the firewall?
2. Allow SSH?
3. Allow Jenkins on port 8080?**

# The Answer:

```bash
hardik@DESKTOP-G15O7OV:/$ sudo ufw allow 22
Rule added
Rule added (v6)
hardik@DESKTOP-G15O7OV:/$ sudo ufw enable
Firewall is active and enabled on system startup
hardik@DESKTOP-G15O7OV:/$ sudo ufw default deny incoming
Default incoming policy changed to 'deny'
hardik@DESKTOP-G15O7OV:/$ sudo ufw default allow outgoing
Default outgoing policy changed to 'allow'
hardik@DESKTOP-G15O7OV:/$ sudo ufw allow 8080
Rule added
Rule added (v6)
hardik@DESKTOP-G15O7OV:/$ sudo ufw status
Status: active
```
