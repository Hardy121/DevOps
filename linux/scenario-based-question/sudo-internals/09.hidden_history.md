**The Scenario:**

You are a security auditor. You suspect a user has been using `sudo` to run malicious commands, but when you check the user's `.bash_history`, it is empty or doesn't show the commands they ran as root.

**The Question:**

Where is the **internal system log file** (provide the full path) where `sudo` records every command successfully executed by any user? Also, if the user ran `sudo -s` to get a shell, does `sudo` log the individual commands they typed *inside* that root shell?

**The Answer:**

`/var/log/auth.log`

`sudo` only logs the **execution** of the shell (e.g., `sudo -s`). Once the user is *inside* that root shell, `sudo` has finished its job. The commands they type are now just standard shell inputs.
