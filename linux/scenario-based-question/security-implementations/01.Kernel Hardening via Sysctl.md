# Scenario:

You are deploying a web server that is experiencing a small "SYN Flood" (a type of DDoS attack). Your senior engineer tells you to harden the network stack at the **Kernel Layer** without rebooting the server. You need to enable **TCP SYN Cookies** and disable **IP Forwarding** (since this isn't a router).

# The Question:

Which file do you edit to make these kernel changes permanent, and what is the command to apply the changes immediately without a reboot?**

### The Answer: The `sysctl` Power Tool

To harden the kernel without a reboot, we use the `sysctl` interface.

1. **The File:** You edit `/etc/sysctl.conf`.
2. **The Parameters:** To stop those SYN flood attacks and disable routing, you would add these lines:
    - `net.ipv4.tcp_syncookies = 1` (Enables a "cookie" handshake to filter fake traffic).
    - `net.ipv4.ip_forward = 0` (Prevents the server from acting as a gateway/router).
3. **The Command:** To apply these changes **immediately** without restarting the server,02 you run:
`sudo sysctl -p`
