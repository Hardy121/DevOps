# The Scenario:

You have a junior developer who needs to check the logs of a production application located in `/var/log/myapp/`. You don't want to give them the full `root` password, but you also don't want them to be able to *delete* the logs—only read them.

In DevOps, we follow the **Principle of Least Privilege (PoLP)**.

# The Question:

Instead of giving the developer the root password, what specific Linux file do you edit to give a specific user "sudo" rights to run ONLY the `/usr/bin/tail` command, and what is the command you should use to edit that file safely (to prevent syntax errors that could lock everyone out)?

# The Answer:

`sudo visudo`
`username  ALL=(root)  NOPASSWD: /usr/bin/tail /var/log/myapp/*` 

`NOEXEC`
